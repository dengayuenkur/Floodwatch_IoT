Features & Architecture

A deep-dive into every capability of the Secure IoT Flood Monitoring System — from hardware sensing to cloud-based alerting and cybersecurity controls.

Layer 1 & 2

Hardware & Edge Sensing

The system uses a NodeMCU ESP8266 microcontroller paired with a HC-SR04 ultrasonic sensor — a widely available, low-cost sensor perfect for demos and prototyping; mount it above the water, keeping the electronics dry.

3-Reading Average
Three pulses averaged per measurement to eliminate sensor noise and false readings.
LED + Buzzer Indicators
On-device green/yellow/red LEDs and audible buzzer provide local alerting without network dependency.
HTTP Retry Logic
Up to 3 automatic retries with 3-second delay if the server is temporarily unreachable.
Offline Resilience
Continues measuring and operating LEDs/buzzer locally even when WiFi connectivity is lost.

JWT Authentication

8-hour access tokens with role claims (admin/operator/viewer) signed with HS256. Expired-token and invalid-token error handlers return structured JSON — never HTML error pages that could leak stack traces.

Sensor API Key

Every ESP8266 POST must include a pre-shared API key in the X-API-Key header. Mismatches are logged to the AuditLog table with the source IP address for forensic review.

Role-Based Access Control

Admin: full access including user management, audit log, sensor creation. Operator: can acknowledge alerts and submit readings. Viewer: read-only dashboard access. Roles are embedded in JWT claims.

Rate Limiting

Login endpoint: 10 requests/minute. Sensor POST: 200/minute. Default: 300/day, 60/hour. Limits mitigate brute-force login attacks and sensor flooding / DoS attempts.

Audit Logging

Every login, failed auth attempt, user creation, sensor registration, and alert acknowledgement is written to the AuditLog table with event type, user, IP address, and timestamp.

SQL-Injection Safe

All database access goes through SQLAlchemy ORM with parameterised queries — raw SQL strings are never constructed from user input. Column types are explicitly mapped for type coercion safety.
